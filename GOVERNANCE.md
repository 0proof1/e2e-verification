# Governance

e2e-verification currently uses a maintainer-led model. Maintainers review
changes for framework neutrality, safety gates, evidence integrity, cleanup,
redaction, and compatibility.

The project does not currently claim a multi-maintainer bus factor. Until
review and release authority is distributed, users should treat maintainer
availability as an explicit continuity risk. Maintainer onboarding, ownership
of trust-critical modules, and a documented release handoff are governance
milestones rather than assumed capabilities.

Changes to schemas, status semantics, approval behavior, adapter loading, or
publication boundaries require a documented design discussion. Releases follow
semantic versioning. Governance will expand as the maintainer group grows.
