# Roadmap

## 0.2

- Stabilize provider-neutral model slots, staged work plans, escalation rules,
  and evidence-backed agent task packets.
- Keep functional and usability status independent.
- Pilot 1366x768 viewport/full-page capture pairs and mechanical layout checks.
- Add screenshot thumbnails and original-artifact links to HTML reports.

## Trust and Real-World Validation

- Publish reproducible, permission-cleared case studies for at least three
  non-synthetic applications before claiming project-verified maturity.
- Add adversarial, property-based, and fault-injection coverage for redaction,
  approval gates, cleanup, resume, and evidence serialization.
- Commission or publish an independent review of trust-critical modules.
- Distinguish designed, synthetic-verified, project-verified, and
  externally-verified claims in release notes.

## Onboarding and Ecosystem

- Publish maintained Spring, Django, and Rails adapter templates with measured
  setup effort and explicit limitations.
- Add a complete synthetic approved-write/cleanup lifecycle.
- Evaluate opt-in browser engines beyond Chromium without weakening the
  deterministic harness boundary.
- Expand reporting for longitudinal comparison and team-scale triage.
- Grow review and release ownership beyond a single maintainer.

## Pilot Follow-up

- Add separate storage-state fixtures and output roots for editor, settlement,
  and external-contributor roles.
- Add loading, populated, empty, and error data-state fixtures.
- Integrate axe checks and deterministic keyboard/focus traversal evidence.
- Run the bounded dashboard, settlements, and budgets pilot only after a
  permission-cleared product profile is supplied; do not represent the
  synthetic dashboard pilot as that product evidence.
